[H2] 1. 研究对象、目标与证据更新后的两个工作机制
[P] 两个独立模型看到同一类 256-token prompt、同一套 marker tokens 和同一套 count tokens，模型规模完全相同；差别只在于被要求生成什么输出序列。Non-thinking 模型把 prompt 直接压缩成一个 count。CoT 模型则先生成逐项 indexed trace，再回答同一个 count。最初我们把它们分别假设为“并行集合聚合”和“逐项检索循环”；经过第 5–11 节的描述、ablation、clean-to-corrupt patch、MLP feature patch、geometry steering 与 residual transplant 后，现在可以把若干箭头升级为工作机制，同时保留尚未闭合的桥接。
[H3] 工作机制 A · Non-thinking：broad set routing 写入后层 count state
[P] 已支持：最终 <Ans> query 的 Layer-1 broad heads 并行覆盖多个 prompt needles。Position-local ablation 使 final accuracy 大幅下降；donor broad slices 能运输部分 count；mask broad group 又会同步破坏末层 count geometry 与输出。Layer-4 natural residual/centroid transplant 最终可以一比一搬运 donor count。
[P] 当前解释：早期 attention 负责收集集合证据；held-out post-GELU feature patch 进一步证明低/中 count 的一部分 MLP features 能按 donor offset 运输数值，再由 residual/后层离散化为 30 个 count logits。高 count 的 state 成形更晚，且局部 centroid path 有效而全局直线 +1 轴不成立。
[P] 仍缺：还没有一套跨 1–30 与跨层通用的稀疏 feature basis；高 count 的 sparse transport 较弱，也没有证据表明后期 count state 会反向控制早期 broad routing。
[H3] 工作机制 B · CoT：targeted retrieval/progress loop，加上独立 final count state
[P] 已支持的 trace loop：数字 <k> 的 targeted multi-head bundle 定位第 k 个 prompt needle并写出 M_k；marker 后 residual 保存可执行 progress/termination state；Layer-3 MLP 转换 routed evidence，Layer-4 的分布式 feature 子空间把它写成 <k+1> 或 </Think> logit。Progress transplant 还能反向改变下一步 targeted routing，形成当前最清楚的局部双向因果耦合。
[P] 必须分开的 final state：最后一个 marker residual 能改变继续/关闭，却不能独立运输最终 scalar count。Source-conflict 显示最终答案跟 trace pair 数/extent，而非 prompt count、marker identity 或最后 trace 数字；clean Layer-1 MLP output 与 post-MLP residual 已能把 donor scalar 写入 conflict receiver。
[P] 仍缺：trace pair 数每变化 1，<Ans> 绝对位置也变化 2。由于模型使用 learned absolute positions，当前实验还不能区分真正的 trace-length aggregation 与 position lookup；固定少数 trace-readout heads 也不足以解释这座桥。

[P] 实线步骤已有必要性或局部充分性实验支持；虚线桥接表示当前仍未定位完整 circuit。切换模式并逐步播放，可查看每一步读取什么、写入什么，以及证据边界。

[H2] 3. 新术语、数据列与计算公式
[P] 本节固定后文所有图表和表格的计算口径。先定义单个 attention row，再定义跨 query/head 的汇总 score，最后定义输出与因果干预指标。除非特别注明，Layer 和 head 在数据表中使用 0-based 编号；报告标题显示 Layer 1–4。
[H3] 3.1 Attention 记号、query 位置与 token 集合
[H4] 单头 causal attention
[P] 这里 q 是 query 位置，j≤q 是 causal mask 允许访问的 key 位置。因此每一行满足 Aqj≥0 且 ∑j≤qAqj=1；下文的 attention mass 都直接对这一概率行求和。
[H4] 任意 token 类别的 attention mass
[P] 例如 prompt_noise_mass 是 M(U|q)，bos_mass 是 Aq,BOS。类别 mass 只描述 attention 权重放在哪里，不直接等于该位置对输出的因果贡献。
[H3] 3.2 本报告使用的 attention-head scores
[P] 该 head 投向全部 prompt needle 位置的总 attention mass。只测“看了多少 needle”，不测是否均匀。
[P] 条件在“attention 已落入 needle 子集”后，分布覆盖多个 needles 的均匀程度。n≤1 或 needle mass=0 时定义为 0。
[P] 同时奖励“总 mass 在 needles 上”与“在多个 needles 间广泛分布”；范围 [0,1]，用于 direct_broad head 排名。
[P] 从数字 query <k> 到 prompt 中按位置排序的第 k 个 needle 的 raw mass；这是 targeted_retrieval 排名真正使用的 score。
[P] 只在 prompt needle 子集内判断最大 attention 是否落在 matching needle；不要求它是整个上下文的 top-1。
[P] 条件在“投向 needle 的 mass”内部，有多大比例给了 matching needle。它可以接近 1，但 raw k-to-k mass 仍很低。
[P] 最终答案 query 投向全部 trace marker tokens M1…Mn 的总 mass；不包括 trace 数字 tokens，用于 trace_readout head 排名。
[P] 当前 marker query 对下一 prompt needle 的 raw mass；用于寻找可能参与 successor/next-retrieval 准备的 heads。
[H3] 3.3 Logit、候选集合、margin 与概率
[H4] Logit 与候选集合内的 softmax
[P] zq,t 是 query 位置 q 对“下一 token 为 t”给出的未归一化分数。候选集合 C 由指标指定；改变 C 会改变概率与 margin 的解释。
[H4] Target logit margin
[P] margin>0 表示正确 token 胜过候选集合内所有竞争者；margin=0 是并列；margin<0 表示至少一个竞争者更高。它测相对证据强度，不是概率，也不自动等于 full-vocabulary accuracy。
[H4] Expected count
[P] 先把 logits 限制到 30 个 number tokens 并重新归一化。E[C] 是 logits 分布的连续位置，不保证是整数，也不是一次实际生成的 token。

[TH] 指标
[TH] 平均单位
[TH] 严格定义
[TH] 阅读注意

[H4] Residual 与 count centroid
[P] h 表示 query token 经过 Layer ℓ 后的 residual-stream 向量，μ 是独立样本上同一 exact count 的均值。Residual transplant 替换完整 h 或 μ 后只运行剩余 Layers；它不是统计学“预测残差”。

[P] 若 count manifold 弯曲，不同 δc 会不平行，单一方向 d 即使可读也未必能稳定 steering。
[H2] 4. 学习动态：不同 count 区间何时达到 99%
[P] 本节先把三条 accuracy 曲线的条件固定下来，再比较学习速度。v10 的 gold count 是 1–30；没有 count=0 样本。所有“区间 accuracy”都先在每个 exact count 内求准确率，再对区间包含的十个（或全范围三十个）exact counts 等权平均，因此不会让某个 count 因样本数较多而占更大权重。
[P] 输入什么：给定完整正确前缀。Non-thinking 直接给到 <Ans>；thinking 还会给定完整正确的 indexed trace，再到 <Ans>。
[P] 怎么算：读取 <Ans> 位置预测下一 token 的 logits，只在 30 个 count-number tokens 中取 argmax；等于 gold count 记 1。
[P] 测量什么：最终答案 readout 是否已经学会。对 thinking 来说，它不要求模型自己生成 trace，因此不是端到端 CoT accuracy。
[P] 输入什么：只提供 prompt 和该模型的格式起点，后续 token 全部由模型逐步生成。
[P] 怎么算：解析自由生成结果中的最终 count token；它等于 gold count 记 1，否则记 0。
[P] 测量什么：端到端最终答案。Thinking 模型的任何 trace 错误都可能传播到最终 count，因此它同时包含 trace generation 与 answer readout 两个环节。
[P] 输入什么：与 AR final-count 相同，不提供 gold trace。
[P] 怎么算：生成的完整 <1> M1 ... <n> Mn token 序列必须与 gold trace 逐 token 完全相同；任何 index、marker、长度或关闭位置错误都使整例记 0。
[P] 测量什么：完整 CoT 路径是否无误。它比逐 token marker accuracy 严格得多；non-thinking 没有 trace，所以该指标不适用。
[P] 对每个 gold count c，先计算该 count 内的样本准确率；随后对所选 count 集合中的这些准确率做算术平均。Figure 1 使用 1–30 三十个 count；Figure 2 分别使用 1–10、11–20、21–30。
[P] TF 每 500 steps 评估；更昂贵的 AR 与 trace exact 每 1000 steps 评估。因此“首次达到 99%”只精确到对应评估网格，不能解释成连续时间中的精确 crossing。

[H3] 4.2 为什么 thinking 的 final count 先学会，而完整 trace 更慢？
[LI] Teacher forcing 把正确 trace 直接放在答案前面。Thinking 的 TF final-count 只需要把已经正确写出的 trace 汇总为一个 count token；它无需先解决第 k 个 needle 的检索、marker identity、successor index 和停止位置。因此这是较短的 readout 子问题。
[LI] 最终答案只有一个监督 token，trace 却有 O(n) 个必须连续正确的 token。Count=30 时，indexed trace 包含 30 个 index 与 30 个 marker。即使单步正确率已经很高，sequence exact match 仍会把任一步错误放大为整例失败，所以 trace exact 天然晚于局部 token loss 和 TF final-count。
[LI] 实际 loss 分解显示瓶颈在 trace marker，而不是最终 count。Step 500 时，thinking 的 final-count loss 已降到 0.0267，但 trace-marker loss 仍为 1.5503，trace-index loss 为 0.1389。也就是说模型已经能从 gold trace 读出 count，但还不能稳定完成每一步 k-to-k marker retrieval。
[LI] AR final count 随 trace 一起改善，说明早期端到端瓶颈确实位于 trace。当 gold trace 不再提供时，错误会在自回归过程中累积；因此 thinking 的 AR final-count 曲线远晚于 TF 曲线，并与 trace-exact 曲线大体同步。两者后期可能略有分离，因为错误 trace 偶尔仍能产生正确最终 count，反之亦然。
[LI] 这不等于 CoT “计算更慢”。这里比较的是优化样本复杂度与严格 sequence 指标：thinking 多学习了一个长度随 count 增长的结构化输出。后文的 attention、ablation 与 patching 才判断这段 trace 是否形成了不同的计算路径，而不是只增加了训练 token。

[H4] PC1–PC6 explained variance 到底在做什么
[LI] 输入不是单样本 hidden states。对每个 site×Layer，先把相同语义标签的 256 维 residual 求均值。final-answer site 按真实 count n=1,...,30 分组；trace site 按进度 k=1,...,30 分组。于是 final site 得到一个 30×256 的类中心矩阵。
[LI] 只分析类中心之间的方差。从每个类中心减去 30 个类中心的总均值，对中心化矩阵做 SVD/PCA。若第 r 个主成分的特征值为 lambda_r，则 EVR_r = lambda_r / sum_s(lambda_s)；PC1–PC6 cumulative 是前六个 EVR 的和。
[LI] 问题是什么。PC1 很高表示不同 count 的均值主要沿一条轴排列；PC1–PC6 很高表示六维子空间足以保留绝大多数 count-class geometry。它不测同一 count 内样本的离散程度，也不意味着每个 256 维 residual 真的只有六维。
[LI] 每个 panel 独立拟合。不同 site 或不同 Layer 的 PC1 不是同一神经方向，因此可以比较 explained-variance ratio，却不能把两个 panel 的横轴直接视为同一向量。

[H2] 7. Attention-head ablation：broad aggregation 与 targeted retrieval 是否因果必要
[P] 第 5 节只根据 attention 权重提出候选机制。本节进行必要性检验：保持输入、参数、MLP 和其他 heads 不变，只移除指定 attention head 的贡献，再观察模型已经学会的行为是否下降。分析分成四个问题：non-thinking broad heads 是否支撑最终计数；CoT targeted heads 是否支撑逐步 marker retrieval；CoT trace-readout heads 是否支撑最终答案；描述性高分与因果必要性是否真的一致。
[H3] 7.1 实验协议：删掉什么、在哪些 token 上测、accuracy 如何计算
[LI] 评估样本。重新生成每个 exact count 8 个 held-out prompts，共 240 个。每个区间 1–10、11–20、21–30 各含 80 个样本，因此三栏的样本量与类别数完全相同。
[LI] 全局 head mask。对 Layer l 的 head h，把 GPT-2 forward 中对应 head_mask[l,h] 设为 0。该 head 在整条 sequence 的所有 query positions都被关闭，而不是只在 <Ans> 或某一个 <k> 关闭；其他 heads、MLP、embedding 和模型参数不变。因而实验能判断“这个 head 是否整体必要”，但不能单独定位它在哪个 query 上发挥作用。
[LI] 单头消融。16 个 Layer×head 分别单独 mask。报告 accuracy drop = baseline accuracy − masked accuracy；drop=1 表示从 100% 降到 0%，drop=0 表示离散 accuracy 没变。
[LI] 累计消融。先按第 5 节定义的分数给 heads 排名，再依次 mask top-1、top-2、…、top-16。Non-thinking 使用 broad_attention_score；CoT 分别使用 k-to-k raw mass、trace_markers_mass 与 successor score。对照只使用预先固定的随机 head 删除路径，不再使用低分或倒序排名。
[LI] 随机对照与 random-4 mean。淡灰细线逐条显示每个固定随机删除顺序；深灰线是在同一个 mask 数 n 上对所有随机路径取均值，灰色阴影是随机路径的 min–max。紫色水平虚线 random-4 mean 是随机路径在“恰好删 4 个 heads”时剩余 accuracy 的均值，并横向延伸作为易读参照；它不是一条随 n 变化的额外实验曲线。只有 ranked-top 曲线稳定早于随机路径分布下降，才能说明描述性排名含有超出随机删除的机制信息。
[LI] Teacher-forced final-count accuracy。完整 gold prefix 已在输入中；在 <Ans> 位置读取 logits，若 argmax 数字 token 等于 gold count 则正确。CoT 中 gold trace 仍存在于左侧，所以该指标检验“给定正确 trace 后的最终 readout”，不是自由生成整条 trace 的端到端准确率。
[LI] Teacher-forced trace-marker accuracy。对 CoT 的每个数字 <k> query，检查其 next-token argmax 是否为 gold marker M_k，再对所有有效 k queries 求平均。后一步仍看到数据中的 gold 早期 trace token；某一步预测错不会被写回上下文并污染后续步骤。
[LI] Teacher-forced trace-index accuracy。在 <Think> 或前一个 marker M_(k-1) 的 query 上，检查 next-token argmax 是否为数字 <k>。它测 successor/index 生成，与 marker identity retrieval 是不同子任务。
[P] 阅读顺序。下面每个小节都严格按照“实验 → 结果 → 分析”组织。实验段只说明干预和指标；结果段先给图表与观测数值；分析段才讨论这些结果支持或排除什么机制。

[H3] 7.6 Position-local ablation：把“哪个 head”与“在哪个 query 起作用”分开
[H4] 实验
[P] 前面的 global mask 会关闭一枚 head 在整条 sequence 的所有输出，因此任何累计顺序都可能混入 Layer 构成与非目标 token positions 上的上游作用。为把“哪个 head”与“在哪个 query 起作用”分开，本实验直接 hook 每层 attention 的 c_proj 输入；这个张量仍按四个 head slices 拼接。对选中的 head，只把指定语义 query 行对应的 64 维 slice 置零，其余 token positions、其余 heads、MLP 与参数全部保持原值。
[LI] Non-thinking broad。只在最终 <Ans> query 屏蔽 selected head outputs，测 final-count accuracy；同一 heads 在 prompt token 上的作用不受影响。
[LI] CoT targeted retrieval。只在所有 gold trace 数字 <k> queries 屏蔽 selected head outputs，测下一 token 是否为正确 M_k 的 trace-marker accuracy；marker、successor 和最终 <Ans> rows 不直接被 mask。

[LI] 严格同层随机对照。same-layer random 在累计第 t 步使用与 ranked-top 第 t 步相同的 Layer，只在该 Layer 内随机选择尚未删除的 head。因此每个 prefix 的 Layer 组成与 ranked-top 完全相同，差异不能再归因于“一个顺序先删 Layer 1、另一个先删 Layer 3/4”。图中逐条显示固定随机路径、逐步均值和 min–max；紫色 random-4 mean 表示同层随机路径在恰好删 4 个 heads 时的平均剩余 accuracy。

[H3] 7.8 本节结论与下一步
[H3] Non-thinking
[P] 当前支持：Layer 1 broad heads 既有描述性 prompt-wide attention，也有强单头/累计必要性；它们很可能是直接 set aggregation 的关键入口。
[P] 仍缺：global mask 不能证明这些 heads 把“count 数值”写到了哪里。第 8 节需要在 <Ans> 局部 patch broad-head output，并测 hidden-state/count logits 是否随 donor count 搬运。
[H3] CoT
[P] 当前支持：k-to-k heads 作为一个组对 marker trace 必要，但单头可替代；targeted retrieval 与 final trace readout 是不同阶段、不同排名的多头 circuit。
[P] 仍缺：teacher forcing 隔断了局部 trace 错误向最终答案的级联。第 8 节应分别在 <k> 和 <Ans> 做局部 activation patch，并使用 marker-identity margin、next-index margin 与 final-count margin 区分检索、successor 和 readout。
[H2] 8. Attention-head patching：候选 head output 是否具有局部因果充分性
[P] 第 7 节的 ablation 问的是“移除某个组件会不会坏”，属于必要性证据；本节反过来问“只把 clean/donor 的这个组件放进 corrupt/receiver run，能不能把缺失的信息带回来”，属于局部充分性与信息运输证据。所有 patch 都发生在 attention 的 c_proj 之前：四枚 head 的 value-weighted outputs 仍是四段独立的 64 维 slices；只替换指定 Layer×head、指定 semantic query 的 slice，其他 token rows、其他 heads、MLP、residual stream 和参数保持 receiver/corrupt 值。
[H3] 8.1 共用实验定义：clean/corrupt、donor/receiver 与两种结果量
[LI] Marker-identity clean-to-corrupt patch。clean 与 corrupt 的 256-token prompt、needle 位置、count、trace 数字和全部上下文长度相同；唯一变化是目标第 k 个 prompt needle 的 marker identity。clean run 需要预测原 marker，corrupt run 在同一位置改成另一 marker。我们在同一个 CoT 数字 <k> query，把 clean 的 selected head-output slices 放进 corrupt run。
[LI] Clean-marker logit margin。在 <k> query 的 next-token logits 中，以 clean marker 的 logit 减去其余 marker token 中最大 logit。margin 为正表示 clean marker 胜过所有 marker competitors；为负表示模型更偏向某个错误 marker。
[LI] Normalized recovery。(patched margin − corrupt margin) / (clean margin − corrupt margin)。0 表示 patch 没有把 corrupt 状态拉向 clean；1 表示恢复到 clean margin；大于 1 是过度恢复；小于 0 表示比 corrupt 更差。先对每个 clean/corrupt pair 计算，再在 count 区间内平均。
[LI] Nested donor-to-receiver patch。receiver count 为 n，donor count 为 m；两者共享同一 noise sequence，且较小 count 的 needle 集合严格是较大 count 的子集。合法 offset 为 m−n ∈ ±{1,2,3,5,10}。在各自最终 <Ans> query，把 donor selected head slices 放进 receiver；CoT 因 trace 长度不同，绝对 position 可以不同，但 semantic query 都是最终 answer readout。
[LI] Expected count 与 transport slope。只在 count-token logits 上做 softmax，计算 E[count]=Σ_c c·p(c)。每个 pair 的 causal shift 是 E[count]_patched − E[count]_receiver；在每个 count 区间拟合 shift = a + b(m−n)。纵轴 slope b=1 表示 donor offset 被一比一搬到输出，b=0 表示没有系统 count transport。
[LI] 候选与随机对照。蓝线按目标机制的描述性 score 从高到低 patch：CoT retrieval 使用 k-to-k mass，non-thinking 使用 broad score，CoT final readout 使用 trace-marker mass。淡灰细线是三条固定随机 head 顺序；深灰是逐 top-n 随机均值，阴影是 min–max；紫色水平线只是 random top-4 mean 的易读参照。低分/bottom 排名不再作为对照。
[LI] Successor/stop 的 nested pair。对每个 k=1,…,29，构造 short prompt（count=k）和 long prompt（count=k+1）。两者共享同一 noise sequence、前 k 个 needle 的位置与 marker identity；long 只在更晚位置多一个 needle。因此从 <Think> 到第 k 个 marker M_k 的 teacher-forced trace 完全相同，差异只在 prompt 是否还存在第 k+1 个 needle。
[LI] Successor query 与双向 margin。patch row 是 marker M_k 本身：其 next-token decision 在 long prompt 应为 <k+1>，在 short prompt 应为 </Think>。Continue margin 定义为 z(<k+1>)−z(</Think>)；close margin 反向定义为 z(</Think>)−z(<k+1>)。两者均以 margin>0 记为目标决定正确。
[LI] Successor controls。successor_top 按 marker query 指向下一 prompt needle 的 raw mass 排序；targeted_top 沿用数字 <k> 的 k-to-k retrieval 排名，检验两个阶段是否共用 heads；wrong donor row 使用相同 successor heads，却把 donor 前一行数字 <k> 的 slice 贴到 receiver 的 M_k row，检验结果是否真的依赖 marker-query activation；另有四条固定随机 head 顺序。

[H3] 8.3 CoT successor/stop：在 marker Mk query 运输“继续还是关闭”证据
[H4] 实验
[P] 本实验专门补上 Figure 5 没有覆盖的下一步：模型已经在数字 <k> 后检索并生成了 marker M_k，此时它如何决定再输出 <k+1>，还是用 </Think> 关闭 trace。对每个 k=1,…,29，我们构造一对严格 nested prompts：short receiver 含 k 个 needles，long donor 含 k+1 个 needles；二者共享全部 256 个 noise tokens、前 k 个 needle 的位置与 marker identity，long 只在更晚的一个空位置新增第 k+1 个 needle。因此两条 teacher-forced sequence 从 <Think> 到 M_k 完全相同，只有 prompt 中“是否还有下一个 needle”不同。
[LI] Continue → close receiver。clean/donor 是 long prompt，正确 next token 为 <k+1>；corrupt/receiver 是 short prompt，正确 next token 为 </Think>。在 receiver 的 M_k query 局部贴入 donor head-output slices，读取 z(<k+1>)−z(</Think>)。若 patch 把短序列拉向“继续”，该 margin 与 target-decision accuracy 应上升。
[LI] Close → continue receiver。clean/donor 改为 short prompt，corrupt/receiver 改为 long prompt；读取反向 margin z(</Think>)−z(<k+1>)。这是独立的反向检验，排除某组 heads 只会无条件增加 index logit 的解释。
[LI] Patch 位置。只在 marker M_k 这一行、attention c_proj 之前替换选中 Layer×head 的 64 维 output slice；前面的数字 <k> row、其他 token rows、未选 heads、MLP 与 residual stream 均保持 receiver 值。
[LI] 四组排序/对照。successor-score ranked 按 M_k query 指向第 k+1 个 prompt needle 的 attention mass 排序；k-to-k targeted ranked 使用上一阶段数字 <k> 到第 k 个 needle 的排名；wrong donor row 使用同一 successor heads，却把 donor 的数字 <k> row 贴到 receiver 的 M_k row；灰线为四条固定随机 head 顺序。
[LI] 结果量。每个 pair 先计算 normalized recovery，再在 1–10、11–20、21–30 三个 receiver count 区间内平均。离散 target-decision accuracy 是 patch 后目标 margin 大于 0 的样本比例；它回答 patch 是否真的翻转了 continue/close 选择，而不只是让 margin 小幅移动。

[H4] 分析
[P] 第一，successor 是分布式局部 circuit，而不是单枚开关 head。单头几乎从不翻转 continue/close；top-4 在中高 count 已明显有效，top-8 才在两种方向上接近完整恢复。这与第 7 节的多头冗余一致，但这里的证据更强：它不是删除后性能下降，而是 selected clean activations 在同一 M_k row 足以把 receiver 的决定向 donor 搬运。

[P] 第三，query row 有信息，但不是绝对隔离。同一 successor heads 若改贴 donor 的上一行数字 <k> activation，top-4 通常显著弱于正确 M_k row，尤其在反向 close patch 与高 count 条件下；然而某些 wrong-row top-8 仍能恢复，说明 continue/close 信息也可能已经分布在相邻 residual/attention rows，不能声称只存在于 marker row。
[H3] 8.4 从 attention evidence 到具体 index logit：MLP 与 residual 的层内转换
[H4] 实验
[P] 本实验继续使用 8.3 节完全相同的 nested short/long pairs 与 marker M_k query，但不再只把整个 head bundle 当成黑箱。每个方向包含 58 对样本（k=1,…,29，每个 k 使用 2 个独立 base sequences），并保留 continue→close 与 close→continue 两个方向。实验仍是单步、teacher-forced 的局部因果分析；按照本节问题范围，没有运行 autoregressive rollout。
[LI] Layer 内计算分解。对每一层，记进入该层的 residual 为 h_pre，attention 输出为 a，则 h_attn = h_pre + a；MLP 输出为 m = MLP(LN₂(h_attn))，层末 residual 为 h_post = h_attn + m。报告把它们依次称为 resid_pre、attn_out、post_attn、mlp_out、post_mlp。
[LI] Residual logit lens。在每个 residual stage 上临时应用模型最终的 ln_f 与 tied unembedding，计算 target token 相对 competitor 的 margin：continue 方向为 z(<k+1>)−z(</Think>)。这回答“到这一 stage 时，目标决定是否已经可被最终读出头线性读取”，而不是声称模型在中间层真的提前退出。
[LI] Component direct-unembedding diagnostic。对 additive component a 或 m 直接乘 unembedding，并计算 clean-minus-corrupt target-aligned margin。由于 component 本身没有经过最终 LayerNorm，这只是定位哪个 component 把证据写向目标 token 的描述性诊断；真正的因果结论来自下一项 patch。
[LI] 关键 MLP mediation 对照。donor Attn; receiver MLP frozen 把 donor attention output 加进 receiver residual，但强制保留 receiver baseline MLP output；donor Attn; native MLP recomputed 使用同一 donor attention output，却让 MLP 对改变后的 h_attn 自然重算。二者 normalized recovery 之差，就是 donor attention evidence 经当前层 MLP 转换带来的局部 mediation gain。
[LI] 其他干预。donor MLP output only 只把 donor MLP additive output 加到 receiver h_attn；donor Attn + donor MLP 同时替换两个 additive components；full donor post-attn state 替换 h_attn 后重算 MLP；full donor post-MLP state 替换整层输出，是该层完整状态充分性的上界/sanity check。

[H3] 8.5 Layer 3–4 MLP intermediate：从 routed evidence 到 index/close logit 的 feature 级因果分解
[H4] 实验
[P] 这里的 MLP feature/neuron 是 GPT-2 MLP 中 c_fc → GELU 后、c_proj 前的一个标量坐标，不是生物神经元。模型的 residual width 为 256，而每层 MLP intermediate width 为 1024；实验只分析 0-based Layer 2–3，即报告中的 Layer 3–4。Query、clean/corrupt pair 和 continue/close margin 与 8.3–8.4 完全相同，仍位于 teacher-forced marker M_k。
[LI] 严格分离 feature 排名与因果评估。对每个 k=1,…,29，使用 2 个 base sequences 拟合 feature 排名，再使用不重叠的 2 个 sequences 做 patch；因此每个方向各有 58 个 fit pairs 和 58 个 held-out eval pairs。
[LI] Feature 排名。对第 i 个 post-GELU feature，先计算 fit pairs 上的 clean−corrupt activation 差，再乘它经 c_proj 与 tied unembedding 对 target-minus-alternative margin 的线性系数。这个乘积的均值是 projected evidence score。它只用于选坐标，不直接当作因果效应。
[LI] Ranked feature replacement。对 support S 中的 feature，把 corrupt receiver 在同一 M_k row 的 activation 坐标替换成 clean donor 值；其他 1024−|S| 个坐标、attention、residual 与后续参数都保留 receiver 值。支持大小依次为 1, 2, 4, 8, 16, 32, 64, 128, 256, 512, 1024。
[LI] Sparse mean-direction transport。在 fit pairs 上求平均 clean−corrupt feature 差方向，只保留 projected-evidence 绝对值最大的 S 个坐标；对 held-out pair，仅运输该 pair 在这个稀疏方向上的投影。它检验“共享低维方向”是否足以替代逐坐标 donor 值。
[LI] 对照。每个 support 使用 4 组固定随机坐标做 size-matched replacement，并另测 random sparse direction。所有结果仍报告逐 pair normalized recovery 与 target-decision accuracy；没有 autoregressive rollout。

[H3] 8.7 Non-thinking MLP intermediate：集合证据如何变成具体 count
[H4] 实验
[P] 这一实验把第 8.6 节的链条再向后推进一步。Receiver prompt 含 n 个 needles，donor prompt 含 m=n±3 个 needles；两者使用 nested prompt，使 receiver 的 needle 集合严格包含于 donor 或反之。我们在最终 <Ans> query、每个 Layer 的 MLP 中间层读取经过激活函数后的 1024 维 post-GELU activation，而不是读取完整 residual。
[LI] Fit split。对每个 feature i 计算 (a_i(m)−a_i(n)) × w_i(m versus n)。其中 a_i 是 post-GELU activation；w_i 是该 feature 经 c_proj 与 tied unembedding 后，对 donor-count logit 相对 receiver-count logit 的线性系数。按跨 fit pairs 的平均 signed evidence 从大到小排序。
[LI] Held-out causal split。Fit 与 evaluation 使用不相交的随机 seeds。在 receiver forward 中，只把排名 top-8 / 32 / 128 / 512 / 1024 个 post-GELU features 替换成 donor 值；其余 features、attention、residual 和后续 Layers 都保留 receiver 值。每个 support size 另采样 4 组等大小 random features。
[LI] Transport slope。对 held-out pairs 回归 ΔE[C] = a + b(m−n)。b=1 表示 patched output 一比一跟随 donor offset；b=0 表示没有数值运输。另报告 R²、donor-vs-receiver logit-margin recovery，以及 argmax 是否真正翻转到 donor count。

[H3] 8.10 CoT final scalar bridge：哪一层把 trace/position evidence 写成可执行 count state？
[H4] 实验
[P] 对同一个 prompt=m, trace=n conflict，先运行 clean donor（prompt 与 trace 都为 m）并缓存最终 <Ans> row 的 activation；再运行 conflict receiver（prompt=m、trace=n，基线输出 n）。我们只在 receiver 的语义 <Ans> row 替换 clean donor activation，测 donor count m 的 logit-margin recovery。由于 donor 与 receiver 的 <Ans> 绝对位置不同，这个 patch 同时问：clean component 是否足以把 donor 的 trace/position-derived count evidence带到 receiver。
[LI] Attention output。替换某一 Layer 完整 attention sublayer 的输出，但保留 receiver MLP 与 residual。
[LI] MLP output。替换该 Layer MLP 投回 residual 的输出；attention 与 MLP 输入仍来自 receiver。
[LI] Post-MLP residual。替换该 Layer 完成 attention、MLP 与 residual additions 后的整条 256 维 state；后续 Layers 继续在 receiver context 中运行。
[LI] Normalized recovery。(patched donor-margin − conflict margin) / (clean donor-margin − conflict margin)。1 表示完全恢复 clean donor margin；另报告 argmax 是否真正翻到 donor target。

[H3] 8.11 本节结论与证据边界
[H3] Non-thinking
[P] 当前支持：early broad heads 既表现出必要性，也能在最终 <Ans> 局部 patch 中运输 donor count；held-out post-GELU feature transplant 又证明低/中 count 的一部分 numeric state 可由特定 MLP feature 子集搬运。因此“broad set aggregation → MLP count conversion → answer state”已有连续的局部因果证据。
[P] 边界：高 count 的 fixed-head 与 sparse-feature transport 都明显变弱，说明 state 分布到更多 heads、Layers 与 residual；尚未得到一套跨 1–30 通用的算术 feature basis。
[H3] CoT
[P] 当前支持：数字 <k> row 的 targeted bundle 恢复 marker identity；marker M_k row 的 successor bundle 运输 next-step evidence；MLP features 写入 continue/close logits。最终答案在冲突中跟 trace pair count/extent，而不是 prompt count、marker identity或最后 trace 数字；clean Layer-1 MLP output 和 post-MLP residual 足以把 donor scalar count 写入 conflict receiver。
[P] 仍缺：trace pair 数与 <Ans> 绝对位置共变，尚未区分语义 trace-length counting 与 learned-position shortcut；也没有把 final scalar bridge 压缩为固定少数 heads 或一根统一 count direction。
[H2] 9. Hidden-state geometry steering：相邻 transplant、非相邻直线与 centroid 曲线
[H3] 9.1 共同设置、状态分解与结果量
[H4] 实验
[P] Centroids 来自独立的 direction-train split，不使用本节干预样本。对 semantic site s、Layer l 和 exact count c，先平均 256 维 residual，得到 mu[s,l,c] = E[h | s,l,count=c]。本节使用 2 个 held-out prompts / exact count，即每个 site 有 60 个 receiver prompts；在三个 sites、四层和所有合法 donor offsets 上共记录 41,184 条真实 patched-forward 行。
[LI] 三个 sites。nonthinking_final_answer 与 thinking_final_answer 都在自然序列的最终 <Ans> query 取 residual。thinking_fixed_trace_answer 则强制所有 prompt 使用同一条 15-step gold-format trace，只有 prompt needle count 与答案标签变化；它用于区分“prompt-derived count geometry 可读”与“该 geometry 真正控制最终答案”。
[LI] Receiver 分解。把 receiver state 写成 h_r = mu_r + epsilon，其中 epsilon 包含该具体 prompt 相对 exact-count 均值的残差。Full transplant 使用 h' = mu_t；delta transport 使用 h' = h_r + (mu_t - mu_r) = mu_t + epsilon。二者若结果相同，说明保留 prompt-specific residual 不妨碍 count 搬运。
[LI] 连续输出。从 count-token logits 计算 E[C] = sum_c c * softmax(z_count)[c]。对每个 panel 回归 Delta E[C] = a + b * Delta c_path；b=1 表示输出一比一跟随指定路径坐标。path-tracking MAE = mean |E_patch[C] - c_path|，越接近 0 越好。
[LI] Layer 含义。“after Layer 1–4”表示替换该 Transformer Layer 输出的 residual。较早层后仍有后续 Layers 可修正或抵消干预；after Layer 4 后只剩最终 LayerNorm 与 tied unembedding，因此更直接检验该 residual 是否已是可执行 count state。
[H3] 9.2 相邻 count：完整 centroid transplant 与 residual-preserving delta
[H4] 实验
[P] 对 receiver count r，分别选择相邻 donor t=r-1 或 r+1。Full transplant 删除 receiver 的 epsilon，直接写入 mu_t；delta transport 只增加局部 centroid 差 mu_t-mu_r，保留 receiver prompt 的其余状态。每个 count 区间独立拟合 transport slope。

[H2] 10. Hidden-state patching：跨 sequence 搬运 count state 与 trace progress
[H3] 10.1 表示空间、配对方式与结果量
[H4] 实验
[LI] 替换的是同一种表示。Donor 与 receiver 都在同一个 Transformer Layer 的 block-output hook 捕获 post-Layer residual，即该 Layer attention、MLP 与两次 residual addition 完成后的 256 维向量；替换后只运行后续 Layers。它不是 embedding、attention value 或 MLP intermediate。
[LI] 跨 sequence pair。每个合法有序 count pair m→n 使用 2 个独立 nested prompt pair；donor 与 receiver 共享 noise base，count 较小者的 needle 集合是较大者的子集。结果仍按 receiver count 1–10、11–20、21–30 分栏。
[LI] 连续结果量。从 30 个 count-token logits 算 E[C]，回归 ΔE[C]=a+b(m−n)。Transport slope b=1 表示 patched 输出一比一随 donor count 移动，b=0 表示没有 count transport；follows donor 则要求 argmax count 恰好等于 m。
[LI] 为什么还保留 centroid transplant。Figure 8A 的 centroid intervention 平均掉 donor prompt identity；后续自然-state patch 则保留一条具体 donor sequence 的全部内容。两者一致时，才能更有把握地说 transported signal 是 count state，而不是某个平均化人工方向。
[H3] 10.2 先验基线：exact-count centroid transplant
[H4] 实验设定
[P] 这个基线先问一个最宽松的问题：同一 count 类别的平均 residual 几何，是否已经足以控制最终 count logits？对每个模型、semantic site、Layer 与 count c，我们用独立 direction split 的样本计算 256 维均值 mu(layer, site, c)。均值样本不包含被测试 receiver，因此 intervention 不会复制同一条 prompt 的内容。
[LI] 正常运行 count=8 的 receiver，并定位它的最终 <Ans> query。
[LI] 若测试 after Layer 2，就把 receiver 在该 token 经过 Layer 2 后的 residual h2(receiver, Ans) 整体替换成 mu(2, Ans, 18)。
[LI] 从 Layer 3 继续运行到 LM head；输入中尚未出现答案 count token，因此输出只能由 patched state 与后续 Layers 决定。
[LI] 从 30 个 count-token logits 计算 argmax count 与 expected count，并对多个 receiver n ← donor m pair 拟合 transport slope。
[P] 图中的 natural donor residual 是更严格的比较：不用类均值，而把一条独立 count=18 donor 在同一 semantic site 的完整 residual 放进 count=8 receiver。Centroid 成功只说明“平均 count geometry 可执行”；natural residual 也成功，才说明单样本状态同样可运输。

[H4] 综合分析
[P] 五类场景共同给出一个稳定分离：marker residual 中可执行的是局部 trace progress，而不是独立的 prompt-total scalar。Natural donor、progress-centroid 与 combined-centroid 三种 intervention 产生相同的第一 index 行为；total-only natural state 与 total-only centroid delta 则完全不动。这使结论不依赖某一个 5→10 例子，也不依赖 donor total 与 progress 恰好同向。
[P] 偏回归也支持同一结论。第一 index 的 b_progress=1、b_total≈0 且 R²=1；final count 的两个系数都接近 0（b_total=0.006、b_progress=-0.012），但其 R²=0.253 较低，因为 27.1% 的 progress-intervention rollout 未自然关闭，线性 final-count 回归只能覆盖成功产生答案的子集。因此 final-count 的更可靠证据是上面的条件命中率与 total-only 零效应，而不是低 R² 回归本身。
[H3] 10.8 本节结论
[H3] Non-thinking
[P] Final <Ans> count state 从早层到后层逐步变得充分，高 count 最慢；after Layer 4 natural residual 能一比一把 donor count 搬到 receiver。
[H3] CoT
[P] 最终 <Ans> 很早就含有可执行 count state；trace final marker 本身不直接等于答案 state，但 marker residual 的后层表示能控制“继续还是结束”。这支持 trace progress/termination 与 final answer count 是相关但不同的 causal states。
[H2] 11. Attention head 与 hidden state 的双向因果联系
[P] 第 5–6 节只描述 head signature 与 count/marker geometry，第 7–10 节分别证明了若干组件的必要性或可控性。本节把两类干预放到同一批样本上，直接检验两条方向：head → downstream state 与 state → downstream head routing。双向不意味着循环反馈；它只要求两个方向都由实际 intervention 支持。
[H3] 11.1 共同实验设计与三个机制链
[H4] 方向 A：head → state
[P] 在语义 query 位置，把指定 attention head 的 c_proj 输入切片置零，只影响该 token 的该 head output；随后在每个 Layer 后读取 residual。Direct/readout 用 exact-count centroid，k-to-k 链用 marker-identity centroid。对照包括 clean、候选 top-1/top-4、同层 control-1 与严格不属于候选 top-4 的 control-4。
[H4] 方向 B：state → head
[P] 先缓存 donor 在某 token 的 residual，再替换 receiver 进入指定 Layer 前的 residual，并读取该层及后续所有 heads 的 attention。Direct/readout 使用不同 count 的 donor→receiver；k-to-k 保持总 count 不变，只改变 trace progress。same-state donor 来自与 receiver 同一语义状态的另一条样本。
[LI] Non-thinking broad chain：<Ans> query 的 Layer-1 broad heads → 后续 scalar-count state。
[LI] CoT targeted chain：trace 数字 <k> query 的 k-to-k heads → 当前 marker-identity state。
[LI] CoT readout chain：最终 <Ans> query 的 trace-readout heads → final scalar-count state。
[H3] 11.2 方向 A：head intervention 是否损伤下游语义 state
[H4] 实验
[P] 对上述三个 query 分别做局部 per-head output mask，并在最终 Layer 后同时测 nearest-centroid accuracy、centroid margin、局部输出 accuracy 与 logit margin。candidate_top4 是按对应描述性 score 排名前四的 heads；noncandidate4_control 与这四头严格不重叠。

[H4] 分析
[P] Direct 模式是非对称链。进入 Layer 1 前的 <Ans> residual 只有 token+position embedding，跨 count 相同，因此 patch 不改变 Layer-1 broad routing。进入 Layer 2–4 前移植 count state 后，低 count 会 100% 采用 donor count；中 count 主要发生连续 margin 搬运，只有 Layer 4 达到 40% exact donor adoption；高 count 更弱。与此同时 broad signature 的平均绝对变化很小。这说明 broad heads 先写入 count state，但后期 count state 并不会反向重建已经发生过的 early broad attention。
[P] CoT final state 对输出充分，但对 readout routing 的控制不稳定。三个区间在四个深度均 100% 采用 donor count；然而候选 readout signature 的变化随 Layer/count 改变方向，高 count 甚至为轻微负值。更合理的解释是 final count residual 可以直接驱动 logits，而 readout heads 是形成该 state 的冗余上游，不是必须被该 state 反向控制的一组固定路由。

[H4] 实验与结果
[H3] Figure 11D. head 导致的 state-margin 损伤是否追踪 output-margin 损伤

[H4] 分析
[P] Direct broad 与 CoT targeted 的大幅 state-margin 损伤通常伴随大幅 output-margin 损伤，符合“head output 改变后续语义 residual，随后影响 logits”的串联解释。CoT readout 的点集中在较小损伤范围，再次显示局部冗余。由于这里没有在同一 intervention 内进一步随机化中介 state，该散点只能作为效应对齐，不能单独声称完成统计 mediation。
[H3] 11.5 综合结论与边界
[H4] Non-thinking
[P] 支持：Layer-1 broad heads → downstream scalar-count state → final logits。后期 count-state transplant 足以搬运低 count 输出。不支持：count state 反向重建早期 broad routing；这是一条前馈、非对称因果链。
[H4] CoT targeted retrieval
[P] 支持：targeted head 组 → marker-identity state，且 progress state → 下一步 targeted routing。二者组成最清楚的局部双向耦合。边界：单头可替代，高 count 下候选外 heads 也重要，circuit 是分布式的。
[H4] CoT final readout
[P] 支持：final scalar state 对 count 输出高度充分。尚缺：固定 readout-head 组与该 state 的强双向耦合；局部 mask 与 routing shift 都显示冗余和异质性。
[H2] 12. 综合机制结论、证据强度与尚缺环节
[H3] Non-thinking
[LI] 描述性 broad heads 位于早期 Layer；分层 mask 检验它们对哪些 count 区间必要。
[LI] Nested donor patch 若在三个区间都有正 slope，说明候选 head slices 不只相关，而能运输部分 count state。
[LI] 完整 centroid transplant 在后层趋近 slope=1，说明 answer-query residual 是足以决定 count 的因果状态。
[LI] Adjacent-mean steering 的区间不稳定性说明该状态不是一根跨 1–30 恒定的标量 +1 方向。
[H3] CoT
[LI] <k> query 的 k-to-k mass 随 count 增大而下降，但 targeted clean patch 相对 bottom/random 的恢复检验其因果功能。
[LI] Targeted retrieval 与 final trace readout 是不同排名；删除 retrieval heads 主要伤 marker trace，不自动等于删除最终 count state。
[LI] 自然 final state 同时含 prompt、trace 内容、trace 长度和位置线索。反事实 fixed-15 control 则把 trace 内容、长度和 <Ans> 位置固定，只让 prompt count/答案 n 改变；它显示 prompt-derived count 仍可读，但分层 transplant 表明该低维几何并非在所有 count 区间都具有一比一因果控制力。
[LI] 现有 trace-progress causality 只覆盖 26–30，尚需低 count 对称实验才能声称统一 successor circuit。
